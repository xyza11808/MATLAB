Copy To Temporary Channel

Email: sigtool@kcl.ac.uk

 

Author: Malcolm Lidierth

Updated: November 2007

Method summary

You can copy data from an existing waveform channel to a temporary channel through the Channels->Copy To Temporary Channel Command.

For episodically sampled channels, only currently valid epochs are copied.

For multiplexed channels, only the currently selected subchannel is copied.

Data for the target channel are copied to a temporary file in your system temp folder. To save the data permanantly, use File->Save(*) or File_>Save As to write a new file.

(*) TODO: Not presently available - use Save As

Options

Channels

Select the source channel and target. By default, unused channels are listed as targets. You may however, select an existing channel (including the source) as the target. In this case all existing data will be overwritten.

Save as Integer

If Save as Integer is selected (the default), data will be stored on disc in integer format. If already in integer format on disc, data from the spource channel will simply be copied. If in floating point format, data will be scaled and offset to use full scale amd converted to signed 16-bit integer.

If Save as Integer is not selected data will be scaled and offset and stored in double precision floating point format on disc. This may be useful if you want to avoid rounding errors due to integer conversion e.g. in applying multiple stage operations on the data. With lengthy data channels, this can lead to slow performance.
